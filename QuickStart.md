## 關鍵概念

### 1. 型別推導與作用域
`:=` 語法用於函式內部，能自動推論型別，提高開發效率。

### 2. Slice 的底層結構
Slice 是對 Array 的動態視窗。當 `append` 操作超過其容量（Capacity）時，Go 會分配新的記憶體空間並搬移資料。**長度（Length）**反應目前元素個數，而**容量（Capacity）**反映底層陣列的大小。

### 3. 指標接收者（Pointer Receivers）
在 Method 中使用 `*` 指標，可以直接修改原始物件的屬性，並避免大型物件複製的開銷。

### 4. 併發哲學
透過 `go` 關鍵字啟動輕量級執行緒（Goroutine），並使用 **Channel** 進行安全的資料傳遞，避免共享記憶體帶來的競爭問題。

### 5. 延遲執行（Defer）
確保資源（如檔案或網路連線）在函式結束前被正確釋放，遵循 **LIFO（後進先出）** 順序。

## 術語清單

- **Goroutine**：Go 內建的輕量級執行緒，初始記憶體佔用極低。
- **Channel**：Goroutine 之間的通訊管道，分為有緩衝（Buffered）與無緩衝（Unbuffered）。
- **Zero Value（零值）**：變數宣告但未賦值時的預設值（例如：int 為 0, string 為 "", slice/map 為 nil）。
- **Struct Embedding**：在結構體中嵌入另一個結構體，實現代碼重用與組合。
- **Panic & Recover**：處理不可恢復錯誤的機制，但在一般業務邏輯中應優先使用 `error`。
